AIF 投資專家團隊 ｜ 2026-06-25 ｜ 手機閱讀版　·　老闆鐵則：不另造平行系統，合進 SIM 既有功能/角色
🎯 一句話結論：量產引擎 ≠ 新系統。它就是 SIM 既有「IC AI建議」子系統（ic_refresh_recs + _ic_score_stock + 信心度）的扶正 + 升級。既有能力已覆蓋引擎骨架 約 70%。要補的只有兩塊大腦：(A) 把 regime 接進融合加權、(B) 把信心度從手調公式換成分條件 base-rate 背書。

關係圖：複用既有(綠) vs 新建缺口(紅)

身體現成（綠塊撐起骨架），缺的是「會看盤的大腦（S1 regime 接線）」與「誠實的信心度（S4 base-rate 背書）」。紅塊＝6 條 R-PROD 缺口。

覆蓋率盤點

  • S2 選股 / S3 訊號：既有覆蓋 ~85%（訊號族幾乎都在）
  • S5 輸出 / 理由 / 追蹤：~90%（IC 子系統現成）
  • S1 regime 接線：~30%（判讀有、未驅動選股）
  • S4 融合與信心度背書：~40%（加分式有、條件式與回測背書無）
  • 加權平均 約 70%

融合心臟：複合桶（避開「找一個指標」死胡同）

鐵錨＝LEDGER 鐵證：所有單一訊號（A1 動量/12 策略/A4 單層/A5 PEAD）全不過閘，唯一過的是 A4 兩層複合 → 融合必須以「複合組合」為單位，不是「單訊號加總」。
  • Layer 0 regime 閘：先判 regime→取策略族白名單+倉位係數。RISK_OFF當日不開新倉。同一訊號在不同盤意義相反（RSI<30：震盪市=便宜、恐慌市=接飛刀）→ regime 是閘門不是加權項。
  • Layer 1 訊號族子分數：每族輸出 [-1,+1]，把現有拍平訊號重組成命名組件。
  • Layer 2 條件式複合：依 regime 加權疊加；把「本次觸發的訊號組合」離散成複合桶（如 TREND_UP × A4∧籌碼確認∧技術多頭）＝可被 base-rate 當「一個單位」回測。低 N / 未過閘的桶→信心度天花板壓低。

信心度＝分條件 base-rate 背書（不再是手調公式）

信心度 = base_rate_lookup(regime, 複合桶) × 即時確認係數
查「該條件切片」的歷史勝率/超額當地基——不要求全期都贏，只要相似 regime/條件的對應時段站得住；大盤（等權公平基準）當參考尺。桶 N<30 或未過閘→硬上限 0.45 標「僅參考」。

MVP（用現成料，最快動起來）

主脊＝唯一過閘的 A4 板塊兩層複合，限 TREND_UP 盤。

在 TREND_UP 盤，引擎每天輸出「屬前段強板塊、且贏過自己板塊、技術翻多、籌碼/情緒不背書」的 Top N 檔，信心度直接掛 A4 的條件式回測號碼（超額 +0.83% CI 下界>0）。

＝把公司唯一的真 alpha，第一次端到端送到老闆面前的可交易推薦。
紀律邊界：A4 是 ALPHA 候選非 keeper（順勢型、2022 破功、單市場、CI 邊際）→ MVP 一律標「候選級信心、RISK_OFF 自動降曝險」；老闆批准才實單。

結語：引擎不是新工程，是把 SIM 既有 IC AI建議扶正成主角——身體現成佔 ~70%，缺「會看盤的大腦（regime）」與「誠實的信心度（base-rate 背書）」。融合以「複合桶」為單位，把「edge 在複合」做成結構，天生避開「找一個通吃指標」的死胡同。研發每畢業一複合→新增一桶→自動進量產。

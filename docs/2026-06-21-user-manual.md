## 1. 系統概觀

### 1.1 架構
- **後端**：FastAPI（Python），SQLite 資料庫 `monitor.db`
- **前端**：單頁應用（SPA），內嵌於 index.html
- **即時行情**：永豐金 Shioaji API（台股）、yfinance（美股）
- **AI 分析**：Claude API（訂閱制或 API 計費）

### 1.2 畫面配置
- **頂部標題列**：系統名稱、市場切換（TW/US）、風控等級 badge、搜尋框
- **左側導航列**：12 個主頁面按鈕（垂直排列）
- **主內容區**：佔滿右側空間
- **底部緊急列**：EXIT_D 停損警示（紅色，出現時不可關閉）

### 1.3 市場切換
頂部標題列有 `TW 台股` / `US 美股` 切換按鈕：
- 切換後，首頁自選股、訊號、持倉皆連動
- K 線圖搜尋欄位也切換對應市場
- 某些功能（如籌碼分析）僅限台股

## 2. 首頁（總覽）

**導航**：點擊左側 🏠 圖示

### 2.1 總經指標列

## 3. K線圖表

**導航**：點擊左側 📈 圖示，或在首頁點擊任一個股

### 3.1 搜尋與選股
- 上方搜尋欄輸入代碼（例：`2330`、`AAPL`）
- 自動載入 K 線圖 + 五檔報價 + 策略標記

### 3.2 時間週期
三個按鈕切換：
- **5分**：5 分鐘 K 線
- **60分**：60 分鐘 K 線
- **日K**：日線

### 3.3 技術指標開關
頂部一排 toggle 切換：
| Toggle | 說明 |
|--------|------|
| 量能 | 成交量柱 |
| MACD | MACD 指標面板 |
| 年線(G1) | 240 日均線 |
| MA5 / MA10 / MA20 / MA60 | 各條均線 |
| 交易 | 交易紀錄標記（買↑賣↓） |

### 3.4 策略標記面板
- 點擊 **📋 策略標記** 展開
- 顯示各訊號在圖上的標記
- **⟳ 同步系統**：當策略設定變更後，重新同步標記

### 3.5 右側面板
1. **五檔委託**：即時 bid/ask 五檔報價表
2. **快速下單**：
   - 選擇波段單 / 當沖單
   - 輸入價格、張數
   - 調整停損滑桿（3%~10%）
   - 買入 / 賣出 按鈕
3. **停損規則**：顯示當前停損設定

### 3.6 NLP 新聞提示
- 當有重大新聞影響持股時，圖表上方出現提示條
- 顯示新聞標題、影響方向

## 5. 交易紀錄

**導航**：點擊左側 ➕ 圖示

### 5.1 市場分頁
- 上方兩個 tab：🇹🇼 **台股** / 🇺🇸 **美股**
- 台股顯示 NT$、單位「張」
- 美股顯示 US$、單位「股」

### 5.2 新增交易
點擊 **新增交易** → 彈出 modal：

| 欄位 | 說明 |
|------|------|
| 股票代碼 | 輸入代碼 |
| 股票名稱 | 自動帶入或手填 |
| 買/賣 | 下拉選 BUY 或 SELL |
| 日期 | 預設今天，可回溯修改 |
| 張數/股數 | 數量 |
| 成交價 | 成交均價 |
| ▼ 交易成本設定 | 展開可調整手續費率(0.1425%)、折扣(60%)、交易稅(0.3%) |
| 備註 | 選填 |

**費用自動計算**：
- 台股手續費 = 成交金額 × 0.001425 × (1 - 折扣)
- 交易稅 = 成交金額 × 0.003（僅賣出）
- 台股成交金額 = 價格 × 張數 × 1000

### 5.3 持倉同步
- **買入（BUY）**：自動在持倉新增或加碼（加權平均成本）
- **賣出（SELL）**：自動減少或平倉持倉

### 5.4 匯入持倉
- 點擊 **📥 匯入持倉**
- 將所有現有持倉自動建立為 BUY 記錄（日期：2026-06-01）
- 首次進入空白頁面時也會自動提示匯入

### 5.5 編輯 / 刪除
- 點擊記錄上的編輯按鈕 → 彈出 modal 修改
- 可修改日期、價格、張數
- 刪除記錄（不影響已同步的持倉）

### 5.6 交易分析
- 點擊 **📊 交易分析** 展開統計
- 已實現損益、個股明細
- 幣別區分（NT$ / US$）

### 5.7 篩選
- 下拉篩選特定股票代碼

## 7. 風控中心

**導航**：點擊左側 🛡 圖示

### 7.1 MACRO_LOCK 全局鎖定
四個總經條件監控：
1. VIX > 35
2. DXY 月漲 > 3%
3. US10Y > 5%
4. 加權指數偏離月線 > 5%

**觸發 ≥ 2 個 → ALERT（部位 30%）**
**觸發 1 個 → CAUTION（部位 60%）**
**無觸發 → NORMAL（部位 100%）**

### 7.2 鎖定操作
- **啟動鎖定**：手動強制 MACRO_LOCK，頂部出現紅色橫幅，禁止所有買入
- **解除鎖定**：恢復正常

### 7.3 風控聯動
- 頂部標題列風控 badge 即時顯示等級（NORMAL/CAUTION/ALERT）
- 風控 ALERT 時，K 線頁買入按鈕自動禁用
- 底部連結可跳轉至「資訊中心 → 總經」查看詳細數據

- 表格：代碼、方向、評分、信心、進場價（含幣別）、評估價、損益%、結果
- **評估結果**：自動抓取當前價格，計算各推薦的實際損益

## 15. 策略實驗室

**導航**：點擊左側 🔬 圖示

### 15.1 功能概述
自動化三層迭代調參系統，對現有策略進行系統化參數優化。

### 15.2 三層優化引擎

| 層次 | 方法 | 說明 |
|------|------|------|
| Layer 1 | Grid Search | 暴力搜索所有參數組合（超過 200 組自動改隨機抽樣） |
| Layer 2 | Bayesian (Optuna) | 在最優區域精搜最佳值，50 trials |
| Layer 3 | AI 分析 | Claude 分析弱點，建議參數微調與邏輯改進 |

### 15.3 操作步驟
1. 選擇要優化的策略（如 BUY_A）
2. 設定回測標的、日期範圍
3. 選擇啟用的層次
4. 點擊 **啟動迭代**
5. 系統自動輪替 Grid → Bayesian → AI，直到收斂

### 15.4 收斂條件（任一觸發即停）
- 連續 3 輪 Sharpe 改善 < 0.05
- 達到最大輪數（預設 20）
- Walk-Forward overfit ratio > 2.0
- 用戶手動停止

### 15.5 結果解讀
- **即時進度**：每輪 Sharpe / 勝率 / MDD 折線圖
- **參數演化**：各參數值隨輪次變化
- **最佳結果**：最佳參數 + 回測績效 + equity curve
- **Walk-Forward**：overfit ratio、各窗口 train/test 對比
- **一鍵採用**：將最佳參數寫入正式策略

## 16. 策略工廠

**導航**：點擊左側 🏭 圖示

### 16.1 功能概述
AI 自主產生**全新**買賣策略程式碼（不同於策略實驗室只調參數），結合多源數據，並累積知識。

### 16.2 四個子分頁

#### 🎛 控制台
啟動策略工廠 session：

| 設定 | 說明 |
|------|------|
| 標的代碼 | 逗號分隔（如 `NVDA,AAPL,2317`） |
| 市場 | TW / US |
| 起迄日期 | 回測驗證時間範圍 |
| 策略數量 | 要生成幾個策略（預設 3） |
| 模式 | explore（探索）/ evolve（演化現有） |

點擊 **啟動** 後：
1. 收集知識庫相關知識
2. LLM 生成策略碼（`evaluate(ctx)` 函數）
3. 驗證（編譯 + dry run + 去重）
4. 回測（動態回測引擎）
5. Walk-Forward 驗證
6. 知識萃取
7. 每個策略重複步驟 2-6

即時 log 面板顯示每步進度。

#### 📚 策略庫
列出所有 AI 生成的策略：
- 策略名稱、類別、Sharpe、勝率、交易數
- 生命週期狀態 badge（draft / testing / validated / promoted / archived）
- 展開可查看完整程式碼
- 操作：晉升 / 歸檔 / 重新回測 / 演化

#### 🧠 知識庫
- 策略工廠累積的結構化知識
- 分類：indicator_combo / performance / market_condition
- 全文搜索（FTS5）
- 知識越多，後續策略生成品質越高

#### 🏆 排行榜
- 所有策略按 Sharpe 排名
- 過濾：全部 / promoted / validated
- 快速比較策略表現

### 16.3 策略生命週期

```
draft → testing → validated → promoted
                            → archived
                            → failed
```

**晉升條件**（全部滿足）：
- Sharpe(OOS) >= 0.5、勝率 >= 45%、MDD <= 25%
- Walk-Forward Consistency >= 60%、Overfit < 2.5
- 交易次數 >= 20

**退場條件**（任一觸發）：
- 近 3 月 Sharpe < 0.2 或 MDD > 35%
- 連虧 >= 3 月
- 無訊號 > 60 天

### 16.4 AI 策略介面規格

所有 AI 策略遵循統一介面：

```python
def evaluate(ctx) -> dict:
    # ctx 包含：
    # - price: 當前收盤價
    # - volume: 成交量
    # - history: 近 N 日 OHLCV list
    # - indicators: {ma5,ma10,ma20,ma60,ma120,ma240,rsi,dif,macd,k,d,bb_upper,bb_mid,bb_lower,atr}
    # - macro: {vix,dxy,btc,eth,es,nq}
    # - date: 當前日期字串
    
    return {
        "signal": 1,       # 1=BUY, -1=SELL, 0=HOLD
        "strength": 0.85,  # 0~1 信心度
        "reason": "..."    # 觸發原因
    }
```

## 17. LLM Queue Handoff

### 17.1 是什麼
策略工廠需要 LLM 生成策略碼，但 `claude -p` CLI 與活躍的 Claude Code session 衝突。
Handoff 機制讓 Claude Code agent 直接充當 LLM，零衝突。

### 17.2 運作方式
1. 策略工廠嘗試直連 LLM（subscription → API）
2. 都失敗時，自動將 prompt 存入 `sf_llm_queue` 資料表
3. Controller 每 5 秒 poll 等待回應（最多 30 分鐘）
4. **Claude Code agent** 或 **sf_agent.py** 讀取 queue、生成回應、回填
5. Controller 立即收到回應，繼續流程

### 17.3 使用方式

#### 在 Claude Code 對話中
直接告訴 Claude：「幫我跑策略工廠研究 NVDA」，Claude 會：
- 啟動 factory session
- 自動監聽 LLM queue
- 生成策略碼並回填
- 回報完整結果

#### 用獨立腳本
```bash
# 先啟動 session
curl -X POST http://localhost:8765/api/sf/session/start \
  -H "Content-Type: application/json" \
  -H "X-API-Token: $TOKEN" \
  -d '{"codes":["NVDA"],"market":"US","start":"2023-01-01","end":"2025-01-01","num_strategies":3}'

# 再啟動 agent
python sf_agent.py <session_id>
```

### 17.4 Queue API

| 端點 | 說明 |
|------|------|
| `GET /api/sf/llm-queue` | 取得待處理項目（status=pending） |
| `POST /api/sf/llm-queue/{id}/respond` | 回填 LLM 回應 |

Queue item 結構：
```json
{
  "id": 5,
  "session_id": 11,
  "purpose": "generate",   // generate / fix / knowledge
  "prompt": "...",
  "status": "pending"
}
```

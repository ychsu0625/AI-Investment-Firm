# v4 UI 設計規範（Design Spec）

**日期**：2026-06-24
**性質**：老闆定調「UI 一起優化美化、參考市面專業軟體、深色主題為主」。本規範是 tool-team 重建 6 區時的**統一設計依據**，每個 zone build 都要遵循，避免各做各的。

---

## 1. 設計方向

**目標**：專業金融交易軟體級的質感。資訊密度高、層級清楚、冷靜專業、深色主題為主。

**參考標竿**（擷取各自強項）：
| 軟體 | 借鏡點 |
|------|--------|
| **TradingView** | 深色配色、K線/圖表質感、資訊密度與留白平衡 |
| **Bloomberg Terminal** | 高密度數據排版、專業冷靜感、等寬數字對齊 |
| **Koyfin** | 現代化金融儀表板、卡片/表格設計、配色克制 |
| **Thinkorswim / IBKR TWS** | 交易介面的操作密度與即時資料呈現 |
| **Linear / Vercel** | 現代 UI 精緻度：間距系統、字體、微互動、flat 無噪點 |

---

## 2. 色彩系統（深色 tokens）

```
背景階層（由深到淺）
--bg-0   #0d1117   頁面底
--bg-1   #161b22   卡片/面板
--bg-2   #1c2128   卡片內分區 / hover
--bg-3   #21262d   邊框感分隔

文字階層
--text-0 #e6edf3   主要
--text-1 #9aa4b2   次要
--text-2 #6e7681   弱化/標籤

語意色（漲跌 — 色盲友善，避免純紅綠）
--up     #2ea043   漲（綠）
--down   #f85149   跌（紅）
--flat   #6e7681   平

強調 / 狀態
--accent #2f81f7   主強調（連結/選中）
--warn   #d29922   警示
規制色（regime）：TREND_UP 綠 / MEAN_REVERT 黃 / RISK_OFF 橘 / CRISIS 紅
```

**規則**：所有顏色走 CSS variable（呼應知識庫 Pattern 23：禁止 undefined var）。漲跌色同時用「顏色 + 箭頭/符號」雙編碼，不只靠顏色。

---

## 3. 字體與數據

- **數字一律 tabular（等寬數字）**：`font-variant-numeric: tabular-nums;`——金融數據對齊的關鍵。
- 主字體：系統 sans（`-apple-system, "Segoe UI", system-ui`）；數據可用等寬（`ui-monospace`）強化專業感。
- 金額/百分比：固定小數位、千分位、幣別前綴（NT$ / US$）。null 顯示 `--` 不顯示 0（知識庫 Pattern 19）。
- **可讀性鐵則（手機優先，老闆痛點）**：一般文字 **≥13px**、次要標籤/tag **≥12px**，**全站禁止 <12px**。行高 ≥1.5。手機上要能輕鬆讀，不瞇眼。

---

## 4. 版面原則

- **8px 間距網格**：所有 padding/margin/gap 用 4/8/12/16/24/32。
- **資訊密度高但不雜亂**：用層級（字級、顏色、分隔線）製造秩序，不靠大留白。
- **Flat**：無漸層、無噪點、無重陰影；用 1px 邊框 + 微背景階層做層次。
- **微互動**：hover/active 有 120-160ms transition；可點元素游標/回饋明確。
- **響應式**：Zone → Tab 兩層（不做三層巢狀，呼應 v2 決策）；窄螢幕 tab 橫向可捲動。
- **禁止整頁水平捲動**（老闆痛點：不方便滾動閱讀）：內容自適應寬度、窄螢幕單欄堆疊；表格過寬則限欄或包在獨立捲動容器，不讓整頁左右滑。
- **可點目標 ≥44px**（手機指觸）。長表格分頁或虛擬捲動，不要無止境長捲。

---

## 5. 元件規範（6 區共用）

| 元件 | 規範 |
|------|------|
| **數據卡 KPI card** | 標題(text-2) + 大數字(tabular) + 漲跌(色+箭頭) + 副資訊；bg-1，1px 邊框 |
| **表格** | 表頭 sticky、數字右對齊 tabular、列 hover 高亮、zebra 可選；密度緊湊 |
| **Badge / Chip** | 規制色/狀態色，圓角小、字級小、padding 緊 |
| **Tab** | 底線式 active（accent）；Zone 內 tab 橫向平鋪 |
| **按鈕** | 主要(accent)/次要(邊框)/危險(down)；危險操作二次確認 |
| **Modal** | 置中、背景遮罩、Esc 關閉、焦點管理 |
| **圖表** | 深色背景、grid 弱化、漲跌色一致；K線參考 TradingView 質感 |

---

## 6. 套用到 6 區

每個 zone 重建時：① 先套色彩/字體/間距 token ② 用上述元件規範重做 ③ 與相鄰 zone 視覺一致。**UI 美化與功能重組同一個工單做，不分兩次**（避免做完功能又回頭改樣式）。

---

## 7. 驗收（站 user 角度）

- 深色下文字對比度足夠（WCAG AA）。
- 數字欄位對齊、小數位一致。
- 6 區視覺語言一致（同一套 token）。
- 無 undefined CSS var、無 console error。
- 實際開瀏覽器看，像專業金融軟體，不像 demo。
